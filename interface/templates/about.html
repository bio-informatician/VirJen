Summary  
The VirJen group (NFDI4Microbiota) based at FSU Jena aims to build a cen-  
tral virus database of all viruses[a] for use by the global research community,  
especially virologists, viral ecologists and others. We will do this by adopting or  
adapting existing database infrastructures (e.g. ViPR; https://www.viprbrc.org  
[1]) and using international metadata standards in accordance with the FAIR  
principles. We plan to incorporate virus genomic and metagenomic sequences  
from public repositories e.g. from the European Nucleotide Archive (ENA,  
https://www.ebi.ac.uk/ena), the Sequence Read Archive (SRA, https://www.ncbi.nlm.nih.gov/sra)  
and GenBank/NCBI viruses (https://www.ncbi.nlm.nih.gov/labs/virus [2]). Fur-  
ther, we will oÐer curation, access, visualization, analysis, integration and shar-  
ing of user-uploaded virus data for a range of user access levels, ensuring data  
protection for embargoed or private datasets.

Database structure and storage  
The VirJenDB will be composed of several diÐerent parts, 1) the Application  
Server, which will send and receive data from the Data Servers, which are 2)  
the File Server and 3) the Database Server, and 4) the Web Server, which will  
communicate with the Application Server and display the results of its requests  
on 5) the Web Interface.  
The 2) File Server (which can be one or more clusters/storage sites) will  
contain or reference all of the large data, including the public and private ge-  
nomic sequences, images, etc. The 3) Database Server (which can also be one  
or more clusters/storage sites) will contain all of the user data, including user  
information and user activity logs. Each of these Data Servers will be backed  
up. See Figure 1 for a schematic.

Data access levels and user groups  
The intention is to oÐer services for both publicly available data and private,  
user-uploaded data. Publicly available virus sequence data will be curated and  
made available through the web interface to all website visitors (public user).  
Various analysis tools will be available for use on the public data. There will  
also be an option on the web interface for visitors to register and subsequently  
(as a logged-in user) be able to work on their own data in a private workbench.  
On the private workbench, the user will be able to use a suite of tools on their  
own data as well as public data.  
The back-end (Parts 1 to 4 above) and front-end (Part 5) access will be  
restricted to provide layers of security to protect user data, including the data

owned by users (ie virus data, stored on Part 2) and the data generated by user  
activity (ie user logs, stored on Part 3).

User groups (front-end users only):  
* Guest: view, analyze, visualize, share, download public data * Data owner:  
view, add and edit own private data, add/remove basic users * Basic user: view,  
add own data  
Private workbench:  
Logged in users will be able to upload their own virus data and metadata within  
one Project. With one or more projects, the user will be able to apply the  
tools on the VirJenDB platform to view, analyze, visualize, share, curate and  
download her own data from one or more projects.

Team Roles  
Currently, the NFDI4Microbiota-funded VirJenDB team is composed of the PI  
(Prof. Dr. Manja Marz), the Data Curator/Team Lead (Dr. Noriko Cassman)  
and the Developer (Shahram Saghaei, MSc.). The latter two roles correspond  
respectively to the Data Curator and Admin groups. There remains at least  
one role to be Ñlled, namely that of the Front-end developer, corresponding  
with the Front-end Developer developer group. Further, the Collaborators will  
contribute to tool development and will work closely with the Admin to add  
tools to the VirJenDB platform.
